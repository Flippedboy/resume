# AI大模型开发工程师个人简历网站 - 需求拆解文档

## 产品概述

- **产品类型**: 个人简历展示网站
- **场景类型**: <scene_type>prototype-app</scene_type>
- **目标用户**: 招聘方、HR、技术面试官、行业合作伙伴
- **核心价值**: 以现代科技感的视觉呈现，全方位展示AI大模型开发工程师的专业能力、项目经验与技术栈，塑造高端技术人才形象
- **界面语言**: 中文
- **主题偏好**: 深色主题（蓝紫色科技风）
- **导航模式**: 锚点导航
- **导航布局**: Topbar（顶部固定）

## 页面布局建议

- **布局模式**: 单栏垂直滚动 + 居中内容容器（最大宽度约束）—— 简历为阅读型内容，单栏布局符合自上而下的阅读习惯
- **视觉重心**: 内容 —— 以信息层次和排版节奏引导阅读，首屏 Hero 区突出姓名与职位建立第一印象
- **结果承载区**: 不适用（本页面为纯展示型，无用户输入/生成结果）
- **响应式策略**: 桌面端居中宽容器（约 1100px），移动端全宽自适应，项目卡片在小屏堆叠为单列

## 功能列表

> **说明**：以下为各内容区块的核心功能/展示要点

- **顶部导航栏**:
  - **页面目标**: 提供快速锚点跳转，提升长页面浏览效率
  - **功能点**:
    - **固定顶部导航**: 滚动时导航栏保持在顶部，背景由透明渐变为半透明深色毛玻璃
    - **平滑滚动跳转**: 点击导航项平滑滚动至对应区块，滚动过程中高亮当前所在区块的导航项
    - **移动端汉堡菜单**: 小屏幕下导航项折叠为汉堡按钮，点击展开/收起

- **顶部个人介绍区 (Hero)**:
  - **页面目标**: 建立第一印象，快速传递身份与核心定位
  - **功能点**:
    - **头像与身份展示**: 圆形头像占位（带科技感渐变边框）+ 姓名 + 职位标签 + 一句话个人简介
    - **联系方式展示**: 邮箱、电话、GitHub、LinkedIn 等图标链接，hover 时有发光动效
    - **首屏渐入动画**: 页面加载时头像、姓名、简介依次淡入上移，营造高级感入场效果

- **专业技能区 (Skills)**:
  - **页面目标**: 结构化展示技术栈与熟练度
  - **功能点**:
    - **技能分类展示**: 按「大模型技术」「编程语言」「深度学习框架」「NLP 技术」等分类呈现
    - **熟练度进度条**: 每项技能配进度条，滚动进入视口时进度条从 0 动画增长到目标值
    - **技能标签云**: 辅助以彩色标签形式展示高频技术关键词，hover 时放大并发光

- **工作经历区 (Experience)**:
  - **页面目标**: 以时间线形式清晰呈现职业发展轨迹
  - **功能点**:
    - **垂直时间线布局**: 左侧时间轴 + 右侧经历卡片，时间节点带发光圆点标记
    - **滚动渐入动画**: 每个经历卡片随滚动依次从右侧滑入淡入
    - **经历详情展示**: 每张卡片含公司名称、职位、任职时间、工作描述要点（bullet 列表）

- **项目经验区 (Projects)**:
  - **页面目标**: 卡片式展示核心项目，突出 AI/大模型相关成果
  - **功能点**:
    - **项目卡片网格**: 响应式网格布局，桌面端 2 列，移动端 1 列
    - **卡片悬停动效**: hover 时卡片上浮 + 阴影加深 + 边框发光，体现交互质感
    - **项目信息完整展示**: 每张卡片含项目名称、技术栈标签、项目描述、核心成果/亮点

- **教育背景区 (Education)**:
  - **页面目标**: 简洁展示学历与教育经历
  - **功能点**:
    - **教育经历卡片**: 学校名称、专业、学位、时间、简要描述（如研究方向、GPA）
    - **滚动渐入**: 进入视口时淡入上移动画

- **荣誉证书区 (Awards)**:
  - **页面目标**: 展示专业认证与荣誉奖项，增强可信度
  - **功能点**:
    - **证书/奖项卡片列表**: 证书名称、颁发机构、获得时间，配图标增强视觉识别
    - **悬停高亮**: hover 时卡片背景微亮 + 图标缩放动效

# UI 设计指南

## 1. 设计推导依据

- **参考意图**: Free —— 无参考材料，按 AI 大模型工程师简历语义自主设计
- **核心情绪 / 应用类型**: 精密、克制、技术权威感的个人品牌 Landing
- **独特记忆点**: 左侧固定的细蓝紫光带贯穿整页，随滚动在对应章节点亮，模拟神经网络激活路径，呼应大模型主题

## 2. Art Direction

- **方向名**: 深空神经控制台
- **Design Style**: Modern Dark + Terminal 几何点缀 —— 深色基底营造专注感，蓝紫渐变呼应 AI 模型训练的视觉联想，等宽字体细节强化工程师身份
- **DNA 参数**: 圆角 subtle (rounded-md) / 阴影 subtle 内发光 / 间距 standard (gap-4 / p-6) / 字体方向：无衬线正文 + 等宽数据 / 装饰手法：细光带、点阵网格、渐变描边
- **应用类型**: Landing —— 单页纵向叙事，章节锚点导航

## 3. Color System

**色彩关系**: 深墨蓝黑基底 + 冷蓝主色 + 紫罗兰强调 + 低饱和灰蓝边界，整体偏冷调科技感
**配色设计理由**: 深色背景降低阅读疲劳、突出代码与技术内容；冷蓝作为 primary 承担主交互与导航激活；紫罗兰 accent 用于技能标签、项目高亮和渐变过渡，制造 AI 联想但不喧宾夺主
**主色推导**: 从大模型训练可视化的蓝紫光谱中提取冷蓝 `hsl(210 100% 60%)` 作为主色，向紫偏移 30° 得到 accent 紫罗兰，保持同一明度与饱和度区间
**使用比例**: 70% 深色中性 / 22% 蓝紫辅助 / 8% primary；primary 仅用于 CTA、当前导航锚点、关键数据高亮

| 角色 | CSS 变量 | Tailwind Class | HSL 值 | 设计说明 |
|---|---|---|---|---|
| bg | `--background` | `bg-background` | hsl(222 47% 6%) | 页面背景，深空墨蓝 |
| card | `--card` | `bg-card` | hsl(222 32% 10%) | 卡片与区块承载面 |
| text | `--foreground` | `text-foreground` | hsl(210 20% 95%) | 标题与正文 |
| textMuted | `--muted-foreground` | `text-muted-foreground` | hsl(215 15% 60%) | 辅助说明、日期、元信息 |
| primary | `--primary` | `bg-primary` / `text-primary` | hsl(210 100% 60%) | 主交互、激活导航、关键数据 |
| primaryForeground | `--primary-foreground` | `text-primary-foreground` | hsl(210 100% 98%) | primary 上的文字 |
| accent | `--accent` | `bg-accent` | hsl(265 85% 65%) | 技能标签、项目高亮、渐变终点 |
| accentForeground | `--accent-foreground` | `text-accent-foreground` | hsl(265 100% 97%) | accent 上的文字 |
| border | `--border` | `border-border` | hsl(220 18% 18%) | 卡片边界、分隔线、输入框 |

**语义色提示**: 成功色 `hsl(160 70% 45%)` 三态：bg `hsl(160 70% 12%)` / border `hsl(160 60% 25%)` / text `hsl(160 70% 65%)`；警告色 `hsl(35 90% 55%)` 三态：bg `hsl(35 60% 12%)` / border `hsl(35 60% 25%)` / text `hsl(35 90% 70%)`；均向冷调偏移，饱和度与 primary 对齐 ±10%

## 4. 字体与节奏

- **font-display**: Space Grotesk —— 几何感强、字形紧凑，体现工程师的精密与现代感
- **font-body**: Noto Sans SC + IBM Plex Mono 点缀 —— 中文正文清晰可读，代码与技能关键词使用等宽字体强化技术属性
- **字号**: H1 text-5xl md:text-6xl；H2 text-2xl md:text-3xl；body text-base；muted text-sm；技能标签 text-xs
- **圆角**: 中 (rounded-md) —— 专业克制，避免过度圆润失去技术感

## 5. 全局布局契约

- **Reference Layout Use**: 按需求结构推导，单页 6 区块纵向排列
- **Page / Section Order**: Hero 个人介绍 → 专业技能 → 工作经历 → 项目经验 → 教育背景 → 荣誉证书
- **Standard Content Zone**: `max-w-5xl mx-auto`，Landing 级阅读宽度，适合简历叙事
- **Shell / Frame Alignment**: 顶部导航 + 左侧光带装饰与内容区同宽对齐，滚动时光带固定
- **Padding & Rhythm**: `px-4 md:px-6 lg:px-8`，章节间距 `py-16 md:py-24`，保持 8px 倍数
- **Full-bleed Zones**: Hero 区背景点阵与渐变光晕全宽，内部姓名与简介受 content zone 约束
- **Local Narrowing**: 教育背景与证书区可收窄为 `max-w-3xl` 居中，突出信息密度
- **Overflow Strategy**: 技能标签行在移动端自动换行，项目卡片在窄屏单列堆叠
- **Flexibility Boundary**: 允许移动端章节 padding 和卡片内边距微调；不允许改变主色、光带装饰和圆角系统

## 6. 视觉与动效

- **装饰**: 点阵网格背景、左侧神经光带、渐变描边卡片
- **阴影/边界**: 轻 —— 卡片用 1px 边框 + 极弱内发光，避免厚重投影
- **动效**: 精致 —— 滚动渐入 (fade-up)、悬停时卡片边框渐变流动、导航锚点平滑高亮、技能条入场填充动画，时长 300-500ms，ease-out

## 7. 组件原则

- 导航链接默认 textMuted，当前章节用 primary 色 + 左侧光点亮起
- 技能标签用 accent 色细边框 + 半透明背景，hover 时背景填充 accent 且文字变亮
- 时间线节点用小圆点 + 垂直线，节点在对应章节滚动到视口时点亮为 primary
- 项目卡片默认 card 底色 + border，hover 时边框渐变从蓝到紫，轻微上移 2px
- 所有可交互元素必须有 `:focus-visible` 外发光环

## 8. Image Direction

- **Image Role**: Hero 背景氛围图 + 头像占位
- **Image Art Direction**: Hero 背景为抽象神经网络可视化——深墨蓝底上流动的蓝紫色节点与连接线，低对比度、暗调、轻微高斯模糊，作为文字背景不抢主体；头像为圆形占位，可用几何渐变 + AI 芯片纹理替代真实照片，保持技术感
- **Image Prompt Keywords**: abstract neural network visualization, dark navy background, glowing blue-violet nodes, thin connecting lines, depth of field, subtle bokeh, cybernetic aesthetic, low contrast, dark mood, 3d render, octane render, minimalist
- **Image Avoidance**: 避免真人商务头像、避免高饱和霓虹赛博朋克拉满、避免电路板/二进制代码等陈词滥调、避免明亮刺眼的科技海报感

## 9. Anti-patterns

- **Split personality**: 不同章节使用不同圆角、阴影或配色；全站统一深空蓝紫系统
- **Neon overload**: 蓝紫渐变铺满背景、按钮、边框、文字，导致视觉噪音；primary + accent 合计不超过 10% 面积
- **Default SaaS drift**: 回到浅色背景 + 通用蓝按钮的默认模板；坚持深色基底 + 神经光带的独特记忆点
- **Invisible interaction**: 只有 hover 动效，忽略键盘 focus 状态；所有交互元素必须有 focus-visible 外发光
- **Mono-hue tyranny**: 主色同时用于按钮、标题、图标、边框、链接；primary 只承担 CTA 与当前导航，其余交给 accent 和中性色
- **Status color drift**: 成功/警告色饱和度过高，在深色底上刺眼；语义色饱和度与 primary 对齐 ±10%
